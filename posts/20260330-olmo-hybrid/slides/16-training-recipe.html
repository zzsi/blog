What Changes in Your Training Stack?

What stays the same (~95%)

  • SAME AdamW optimizer, WSD-S LR schedule
  • SAME Data pipeline (Dolma), 6T tokens
  • SAME Pretrain, mid-train, long-context, post-train
  • SAME Post-training: Think SFT, Instruct SFT, DPO
  • SAME Evaluation suite (OlmoBaseEval)

What's new

  • NEW flash-linear-attention (fla) kernels for GDN layers
  • NEW DroPE for long-context (GDN provides implicit positional info)
  • NEW Shape tuning: 30 heads, adjusted dims to match throughput
  • NEW Doubled mid-training batch size

The hybrid architecture gives big gains with minimal recipe changes. The main tooling difference is the fla library for GDN kernels.
