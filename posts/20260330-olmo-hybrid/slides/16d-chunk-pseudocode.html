Chunk-Parallel: Pseudocode

Schematic view of one DeltaNet layer's forward pass during training

Training loop (one layer)

S = zeros(d_v, d_k) # recurrent state for chunk in chunks: K, V, Q, beta = chunk_projections() # structured factors from rank-1 structure W, U = structured_factors(K, beta, V) # within-chunk outputs: dense matmuls O = Q @ S.T + (Q @ K.T * causal_mask) @ (U - W @ S.T) # boundary update: one step per chunk S = S + (U - W @ S.T).T @ K

Reading the code

W encodes how the chunk transforms incoming state. U encodes what the chunk writes.

U - W @ S.T is the chunk's net update: new writes minus what overlaps with existing memory.

Q @ K.T * causal_mask handles within-chunk token interactions (analogous to attention scores).

Attention: $O(L^2)$ compute, fully parallel (big matmuls).

DeltaNet: $O(L)$ compute, chunk-parallel at training, sequential at decode.

OLMo Hybrid slightly outperforms OLMo 3 in early throughput benchmarks.
