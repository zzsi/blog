Step 2: DeltaNet

Schlag, Irie & Schmidhuber 2021. Lift from vector to matrix memory, apply at every token.

From delta rule to DeltaNet

Lift from weight vector to a memory matrix $S \in \mathbb{R}^{d_v \times d_k}$ mapping keys to values. Now $t$ indexes tokens (not training examples). At each token:

$S_t = S_{t-1} + \beta_t\underbrace{(v_t - S_{t-1}k_t)}_{\text{retrieval error}}\;k_t^\top$

Equivalent (expand and rearrange):

$S_t = \underbrace{S_{t-1}(I - \beta_t k_t k_t^\top)}_{\text{erase along } k_t} + \underbrace{\beta_t \, v_t \, k_t^\top}_{\text{write new}}$

$k_t, v_t, q_t$ are linear projections of input $x_t$ (as in attention). Write uses $k_t$, read uses $q_t$: $\;o_t = S_t \, q_t$.

What is $\beta_t$?

A per-head, data-dependent gate:

$\beta_t = \sigma(w_\beta^\top x_t + b_\beta) \;\in (0, 1)$

Controls how much to erase the old value at key $k_t$ and how strongly to write the new value. Learned via backprop.

Key insight: DeltaNet doesn't just accumulate memories. It edits existing ones by erasing along $k_t$ before writing. This is why retrieval error stays bounded as sequence length grows.

Key limitation: Forgetting only happens along the current key $k_t$. Old memories in other directions persist forever, even if no longer relevant.
