Step 3: Gated DeltaNet

Yang et al., ICLR 2025. Merging Mamba-2's gating with DeltaNet's error-correcting updates.

Recall: DeltaNet

$S_t = S_{t-1}(I - \beta_t k_t k_t^\top) + \beta_t\,v_t\,k_t^\top$

Two changes → Gated DeltaNet

$S_t = \underbrace{\color{#E41159}{\alpha_t}}_{\text{NEW: decay}} \cdot\; S_{t-1}(I - \beta_t k_t k_t^\top) + \underbrace{v_t k_t^\top}_{\text{no more }\beta_t}$
DeltaNetGated DeltaNet
Forgetting Only along key $k_t$ Global decay via $\alpha_t$ on all of $S$
Write strength Tied to erase ($\beta_t v_t k_t^\top$) Decoupled (always full $v_t k_t^\top$)

Note on the write term: The original GDN paper (Table 1) writes $\beta_t v_t k_t^\top$. The OLMo paper writes $v_t k_t^\top$ (absorbing $\beta_t$ into the value projection). The practical effect is the same: erase and write strengths can be learned independently.

How is $\alpha_t$ computed?

A scalar gate, data-dependent, one per head:

$\alpha_t = \exp\!\big(\underbrace{-A}_{\text{learned, per head}} \cdot \underbrace{\mathrm{softplus}(W_\alpha x_t + b)}_{\text{data-dependent, } > 0}\big)$

$\mathrm{softplus}(x) = \log(1 + e^x)$, always positive. Exponent is always negative, so $\alpha_t \in (0, 1)$. Each head's $\alpha_t$ multiplies its entire state $S_{t-1}$ uniformly.

Where does this come from?

Borrowed from Mamba-2's SSM discretization. Start with a continuous-time ODE: $\dot{h} = Ah + Bx$, where $A < 0$ controls how fast the state decays. Discretizing with step size $\Delta$: $h_t = e^{\Delta A} h_{t-1} + \ldots$

Mamba's insight: make $\Delta$ input-dependent ($\Delta_t = \mathrm{softplus}(W x_t + b)$). Now the model learns when to forget. GDN adopts this directly.

Why this helps: In DeltaNet, a memory stored for key $k_{\text{old}}$ persists forever if that key never reappears. With $\alpha_t < 1$, all entries of $S$ shrink each step. Stale memories decay automatically, making room for new ones.
